## 2. What was done today

### 2a. TradingBrain — intelligent coordinator (`strategy_app/brain/`)

A new package added to `strategy_app`. It sits as a gate layer above `DeterministicRuleEngine` and provides three capabilities the engine alone cannot.

**Files created:**

| File | Purpose |
|---|---|
| `brain/plugin.py` | `ContextProvider` ABC, `StrategyPlugin` ABC — the extension contracts |
| `brain/context.py` | `DayContext`, `DayScore`, `SessionCarry`, `FitnessScore` dataclasses |
| `brain/session_memory.py` | JSONL-backed cross-session state (append-only, same pattern as signals/positions) |
| `brain/consensus.py` | `ConsensusGate` — N-of-M strategy agreement before entry |
| `brain/fitness.py` | `StrategyFitnessEvaluator` — maps DayContext to active strategy set + size multipliers |
| `brain/brain.py` | `TradingBrain` orchestrator |
| `brain/providers/daily_features.py` | `DailyFeaturesProvider` — reads `daily_regime_features.json` |
| `brain/providers/llm_stub.py` | `LLMContextProvider` — no-op stub, ready for wiring |
| `tests/test_trading_brain.py` | 35 tests, all passing |

**What the brain does at runtime:**

```
09:00 IST  morning_briefing(date)
           ├─ DailyFeaturesProvider.provide()  → reads daily_regime_features.json
           ├─ LLMContextProvider.provide()     → {} today (no-op)
           ├─ SessionMemory.load_carry()       → yesterday's consecutive losses
           └─ synthesises DayContext (DayScore + size_multiplier + session carry)

09:15 IST  on each entry candidate:
           gate_entry(votes, context)
           ├─ DayScore.AVOID? → block all entries
           └─ ConsensusGate  → require N-of-M strategy agreement

15:15 IST  on_trade_result() updates session counters
           save_session_summary() → appends to session_summary.jsonl
```

**3TC (stop after 3 consecutive losses) — fully wired:**
- `DEFAULT_MAX_CONSECUTIVE_LOSSES = 3` in `strategy_app/constants.py`.
- `RiskManager.consecutive_loss_limit` triggers when hit.
- **New:** `session_memory.jsonl` carries `consecutive_losses_at_close` across calendar days.
  If yesterday ended with 2 losses, today starts with `consecutive_losses = 2` in `RiskContext`.
  The third loss today triggers the pause — previously it reset to zero at midnight.

**DayScore logic:**

| Score | When | Effect |
|---|---|---|
| `CALM` | rv20 < 0.010 + positive SMA slope | Short premium strategies preferred, 1.0× size |
| `NEUTRAL` | No strong signal | All strategies at 0.85× size |
| `VOLATILE` | rv20 > 0.018 | Short premium blocked, other strategies 0.5× |
| `AVOID` | LLM says AVOID, or 3+ losing days streak | No new entries |
| `UNKNOWN` | `daily_regime_features.json` absent | Passthrough — identical to pre-brain behaviour |

**Backward compatibility:** until `daily_regime_features.json` exists, every session runs as `DayScore.UNKNOWN` — no entries are blocked, no behaviour changes. Zero configuration required to use the existing setup.

### 2b. Daily regime feature builder

`ml_pipeline_2/scripts/feature_builder/build_daily_regime_features.py`

Reads futures parquet (or a daily CSV) and writes `daily_regime_features.json`:
- `regime_rv20` — trailing 20-day realised vol from daily log returns
- `regime_dist_sma20` — (spot − SMA20) / SMA20
- `regime_sma20_slope` — 5-day change of SMA20 / SMA20
- `regime_60d_return` — cumulative futures return over 60 days

All values shifted by 1 day to prevent look-ahead.

```bash
# Run nightly at 08:45 IST before session open
python3 -m ml_pipeline_2.scripts.feature_builder.build_daily_regime_features \
    --parquet-root /opt/option_trading/.data/ml_pipeline/parquet_data/snapshots_ml_flat_v3 \
    --output /opt/option_trading/.run/strategy_app/daily_regime_features.json
```

## 6. Risk reminders

1. **No real money should be deployed until:** forward OOS (2025+) audit passes AND broker integration is tested in paper mode for ≥ 5 days with zero discrepancies.

2. **stop_pct = 100 (thesis rule)** means the only hard cap on a losing position is the underlying stop (0.3% adverse futures move). In a genuine tail event (Aug 5 2024 style), this fires multiple times in one session. Always run with a daily-loss circuit breaker active.

3. **3TC is a pause, not a halt.** After 3 consecutive losses, `is_paused = True` — new entries blocked but open positions are managed normally. A win resets the counter. The operator halt (`/api/operator/halt`) is the hard stop.

4. **Playbook V1 is in paper mode.** `playbook_v1_paper_v1` profile is annotated as `rollout_stage=paper`. No real orders. The broker layer does not exist yet.

5. **The brain defaults to UNKNOWN.** Until `daily_regime_features.json` is built and `BRAIN_DAILY_FEATURES_PATH` is configured, the brain gates nothing. This is intentional — safe default.
