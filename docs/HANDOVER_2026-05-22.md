### 2. Critical bug found and fixed: 40% stop was never enforced

**Root cause:** `set_run_context` in `deterministic_rule_engine.py` is called on every snapshot event by the Redis consumer. Event metadata contains only `run_id` ‚Äî no `risk_config`. The old code unconditionally reset `_run_risk_config` to defaults (`stop_loss_pct=None`) on every tick. The engine then fell back to `StrategyVote.proposed_stop_loss_pct` default = `0.40`.

Result: ALL previous replay runs used 40% stop regardless of `_TRADER_MASTER_RISK_CONFIG`.

**Fix (commit `ffd5c83`):** Only overwrite `_run_risk_config` when `risk_config` is explicitly present in metadata. Profile-level config set at startup is now preserved across all ticks.

**Verified:** The `stop_fix_replay` (run `5a8b6684`) shows `stop_loss_pct: 0.20`, `target_pct: 0.70` in DET_DIRECTION positions. Confirmed that Aug 16 trade went from STOP_LOSS -40.1% ‚Üí TIME_STOP -5.9%.

### Decision 2: Multi-layer exit ‚Äî 60% partial + hold runner

**Current:** Exit entire position at 70% target.  
**Problem:** Model predicts 100-pt BN move in 5 min; if it's right the option can easily go +100-150%. We're leaving the "runner" gains on the table.

**Proposed design:**
```
Layer 1 (capital save):  +60% option gain ‚Üí exit 50% of lots  (lock profit, free capital)
Layer 2 (runner):        remaining 50% trails at 6% offset    (let winner run)
Layer 3 (hard target):   +120% on remaining ‚Üí full exit       (or trail fires first)
```

This requires:
- `PositionTracker` / `PositionContext` support for partial lot exit (not implemented yet)
- A "split fill" signal: exit N lots, continue tracking M lots
- Or simpler first version: 2 separate positions of `lots/2` each, different targets

**Files to modify:**
- `strategy_app/position/tracker.py` ‚Äî partial exit logic
- `strategy_app/position/position_factory.py` ‚Äî split-position support
- `strategy_app/risk/config.py` ‚Äî `PositionRiskConfig`: add `partial_exit_pct` / `partial_exit_lots_fraction`
- `strategy_app/engines/profiles.py` ‚Äî add partial exit params to `_TRADER_MASTER_RISK_CONFIG`

**Quick MVP (no partial exit code needed):** Change `target_pct` to `0.60`, disable trailing on exit, add a second run with `target_pct=0.70` to compare. The full multi-layer needs new code.

### Decision 3: Improve direction ‚Äî next priority after entry HPO

**Current direction logic** (`_shadow_direction_from_snapshot`): Multi-signal scorer using:
1. Opening range break (ORB side) ‚Äî weight 2.5
2. VWAP relation ‚Äî weight 2.0
3. ATM option premium momentum ‚Äî weight 1.5
4. PCR change (put-call ratio delta) ‚Äî weight 1.5

**Problem:** DET_DIRECTION WR is 29%. Multi-signal scorer is no better than old simple momentum. Likely reasons:
- Option premium momentum (signal 3) adds noise ‚Äî it's correlated with recent moves, not predictive of next move
- PCR change (signal 4) is noisy intraday
- No temporal context: all signals use the same 5-min window
