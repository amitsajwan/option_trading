### Direction: why the search is over

Six independent tests all converge:
1. Per-member accuracy: all members 48–52% over 37,050 bars (quorum 50.3%)
2. 31-combo ablation grid: only 2/31 positive after cost in BOTH halves, both marginal
3. 2026 OOS: quorum 43.9% — direction INVERTS out-of-sample
4. Order-flow proxies: all ~50% at 1–5 bar horizons over 78,009 bars
5. Direction split on high-prob entry bars: 50/50
6. Live ground truth: 125 trades, zero net-positive days after costs

**Conclusion:** magnitude is predictable; direction is not. The system architecture
should be built around magnitude detection (entry gating for sellers) not direction
prediction (entry side selection for buyers).

## 5. Architecture: The Two Regime Systems

See `docs/TWO_REGIME_SYSTEMS.md` for the full reference. Summary:

There are **two separate "regime" concepts** in the codebase — confusing them caused
months of wrong analysis:

1. **`Regime` enum** (`strategy_app/engines/regime_classifier.py`) — TRENDING / SIDEWAYS / BREAKOUT / AVOID / CHOP / PRE_EXPIRY / EXPIRY / HIGH_VOL / PANIC / DEAD_MARKET. Gates which **entry strategies** are available via the profile map. `AVOID` and `CHOP` → empty entry list = no trades.

2. **`RegimeDirector` quality** (`strategy_app/brain/regime_director.py`) — TREND / MID / CHOP. Gates **direction confidence**. `REGIME_ALLOWED=MID,TREND` is a DIRECTION QUALITY filter, not a Regime-enum filter. Does NOT block entries for Regime-enum BREAKOUT.

`EXIT_STRATEGY_MODE=adaptive` routes by **Regime enum** (not RegimeDirector quality):
BREAKOUT/TRENDING → lottery stack (20% stop), all others → scalper stack.

## 6. Config Gotchas

| Gotcha | Detail |
|---|---|
| `REGIME_DIRECTION_SIGNAL` | `weighted` (graceful, absent=0) vs `combo`/`agreement_lever` (hard-block, absent=ABSTAIN). In 2024 OI/max_pain absent ~46% of bars → combo fires almost never. Use `weighted`. |
| `REGIME_W_MOM=0` | Drops momentum_15m from weighted direction — it is an ANTI-signal (48.1% acc). Set this in live config. |
| `ATR_ENTRY_MIN_PCT` | Default 0.00088 (p90 of live ATR). For 2024 backtests use 0.0006 (2024 vol was lower). The 0.0006 is a knife-edge in backtests — do not treat it as validated. |
| `EXIT_SCALPER_HARD_STOP_PCT` | Only applies to scalper stack. BREAKOUT/TRENDING entries use `LOTTERY_HARD_STOP_PCT`. Both must be set if using `EXIT_STRATEGY_MODE=adaptive`. |
| `.env.compose` | **ONE source of truth**: `/opt/option_trading/.env.compose` (185-line brain config). The 85-line one in `.deploy/runtime-config/` is the old C1 config — it reverts to `EXECUTION_ADAPTER=paper` with `stop_loss_pct=0.25` defaults that bypass the policy stack. |
