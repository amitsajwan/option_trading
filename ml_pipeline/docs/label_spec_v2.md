# Label Spec V2 (T15): Path-Aware Outcomes

> T27 add-on: breakout-aware alternative labels are generated by `ml_pipeline.label_validation` as a parallel validation view (does not replace base CE/PE labels).

T15 extends label generation with intrahorizon path outcomes in addition to fixed-horizon forward returns.

## Inputs

- Feature table (`t04_features.parquet`)
- Options minute bars by day from archive
- Label config (horizon/threshold + excursion gate)
- Exit thresholds for path outcomes:
  - `stop_loss_pct`
  - `take_profit_pct`
  - hold-extension controls

## Trade Window

For decision at minute `t`:

- Entry: `t+1` open
- Horizon exit: `t+H` close (default `H=3`)
- Path window: all bars from `t+1` to `t+H` inclusive

## New Columns (per side)

For both CE and PE:

- `*_tp_hit` (0/1)
- `*_sl_hit` (0/1)
- `*_path_exit_reason` in `{tp, sl, tp_sl_same_bar, time_stop, invalid}`
- `*_first_hit` (same as `*_path_exit_reason`)
- `*_first_hit_offset_min` (0-indexed minute offset from entry, NaN if none)
- `*_tp_price`
- `*_sl_price`
- `*_time_stop_exit` (1 when neither TP nor SL hit in-window)
- `*_hold_extension_eligible` (1 when hold-extension criteria pass at time-stop)

Existing columns (`*_forward_return`, `*_mfe`, `*_mae`, `*_label`) remain unchanged.

## Path Rules

1. Iterate bars from entry to horizon.
2. If both TP and SL are crossed in same bar: `tp_sl_same_bar`.
3. Else first crossed boundary sets reason (`tp` or `sl`).
4. If neither boundary is crossed by horizon: `time_stop`.
5. If entry/exit is not available: `invalid`.

## No-Lookahead Guarantee

Path outcomes only use bars within `[entry_ts, exit_ts]`.
Bars after horizon do not affect labels or path outcomes.

## Outputs

- Labeled parquet (`t05_labeled_features.parquet`) with V2 columns.
- Summary report (`t05_label_report.json`).
- Path report (`t15_label_path_report.json`).
- T27 breakout alternative label parquet/report:
  - `t27_breakout_alternative_labels.parquet`
  - `t27_label_validation_report.json`
  - `t27_label_validation_summary.md`
